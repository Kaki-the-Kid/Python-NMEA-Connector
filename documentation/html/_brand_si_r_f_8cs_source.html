1 /******************************************************************************************
2  * SiRF
3  * The SiRF line of chips support several input sentences that permit
4  * the user to customize the way the chip behaves. In addition SiRF has
5  * a binary protocol that is even more powerful permitting different
6  * implementations to behave entirely differently. However, most
7  * applications do not attempt to customize the behavior so a user will
8  * need to make sure that the any customization is compatible with the
9  * application they are planning to use. There are 5 input sentences
10  * defined that begin with $PSRF which is followed by three digits.
11  * Each sentence takes a fix amount of input fields which must exist,
12  * no null fields, and is terminated with the standard CR/LF sequence.
13  * The checksum is required.
14  *
15  * The sentences 100 and 102 set the serial ports. 100 sets the main
16  * port A while 102 sets the DGPS input port B. 100 has an extra field
17  * that can be used to switch the interface to binary mode. Binary mode
18  * requires 8 bits, 1 stop bit, no parity. There is a command in binary
19  * mode that will switch the interface back to NMEA. Do not use the
20  * NMEA command to switch to binary mode unless you have the ability to
21  * switch it back. You could render your gps inoperative.
22  ******************************************************************************************/
