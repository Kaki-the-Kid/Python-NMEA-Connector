NMEA port

NMEA - National Marine Electronics Association, er det amerikanske standardiseringsorgan for navigationsudstyr som sonar og GPS-systemer. NMEA har eksisteret længe før GPS. I følge NMEAs webside, blev organisationen formet i 1957 af en gruppe electronik forhandlere for at lave en bedre kommunikations model sammen med deres producenter.

I dag efter indførsel af GPS, er NMEA et standard dataformat som bliver understøttet af alle GPS producenter. Formålet med NMEA er at give brugeren muligheden for at mikse hardware og software. NMEA-formatteret GPS data gør livet nemmere for software udviklere med at skrive software for en bred varietet af GPS modtager istedet for at skulle skrive special interface for hver GPS modtager. F.eks. accepterer VisualGPS software (free) NMEA-formatteret data fra enhver GPS modtager og kan vise det grafisk. Uden en standard som NMEA, ville det være være tidskrævende og omkostningsindtensiv at skrive og vedligeholde software.

Det kan være en lille smule forvirende omkring NMEA er at der ret mange typer “NMEA” beskeder. Så på samme måde som der er alle former for GPS modtagere med forskellige egenskaber, er der mange forskellige typer af NMEA beskder kommandoer med forskellige egenskaber. Der udover kan NMEA data transmitioner over forskellige types af grænseflader som: RS-232, USB, Bluetooth, Wi-Fi, UHF og mange andre.

NMEA0183 og NMEA2000 protokoller - datapakker

Information passerer over netværk i segmenter. Hver Datasegmentet bæres i en struktur, der kaldes en pakke . De transport protokoller TCP /IP oprette pakken, og derefter videregive det på Internet Protocol , som tilføjer en header på forsiden, der omfatter den destination IP-adresse. Når du opretter pakken, transport -protokollen tilføjer sin egen overskrift. Dette omfatter det portnummer , som vil få data til en proces, der kører på destinationscomputeren . To transportprotokoller tjener som alternativer : . Transmission Control Protocol eller TCP og User Datagram Protocol - eller UDP

NMEA 0183

De fleste GPS modtagere kan modtage standarden som er kaldet: 0183 version 2. Den standard dikterer en hastighed på 4800 b/s. Nogle modtagere kan også forstå ældre standarder. The oldest standard was 0180 efterfulgt af 0182 which transferred data at 1200 b/s. DEr er også nogle modtagere som forstår tidligere version af 0183 kaldet version 1.5. Nogle Garmin enheder og andre mærker kan sættes til 9600 for NMEA output eller endnu hurtigere, men bør kun bruges hvis man har fastslået at 4800 virker ok og så kan du prøve at sætte den hurtigere. Opsætning så hutigt som muligt giver måske en forbedring af programmers reaktion.

NMEA 0183 definerer et sprog i hvilken geolokalisering systemer uddata. Udgangen skal fortolkes tilbage i geografiske data ved en visning program. Optagelsen enheden og beskueren kommunikerer via netværk ved hjælp portnummer 10110 . Som kommunikation med NMEA -kompatible enheder tendens til at være interaktiv, er forbindelser etableres sædvanligvis på TCP port 10110 , ikke på UDP-port . UDP etablerer ikke en session.

NMEA kommandoer

NMEA består af sætninger, det første ord, kaldet data typen, definerer fortolkningen af resten af sætningen. Hver data type har sin egen unikke fortolkning defineret i NMEA standarden.

GGA sætningen (vist nedenunder) shows an example that provides essential fix data. Other sentences may repeat some of the same information but will also supply new data. Whatever device or program that reads the data can watch for the data sentence that it is interested in and simply ignore other sentences that is doesn't care about. In the NMEA standard there are no commands to indicate that the gps should do something different. Instead each receiver just sends all of the data and expects much of it to be ignored. Some receivers have commands inside the unit that can select a subset of all the sentences or, in some cases, even the individual sentences to send. There is no way to indicate anything back to the unit as to whether the sentence is being read correctly or to request a re-send of some data you didn't get. Instead the receiving unit just checks the checksum and ignores the data if the checksum is bad figuring the data will be sent again sometime later.

In addition some GPS receivers can mimic Loran-C receivers by outputing the LC prefix in some of their messages so that they can be used to interface to equipment that is expecting this prefix instead of the GP one.

The last version 2 iteration of the NMEA standard was 2.3. It added a mode indicator to several sentences which is used to indicate the kind of fix the receiver currently has. This indication is part of the signal integrity information needed by the FAA. The value can be A=autonomous, D=differential, E=Estimated, N=not valid, S=Simulator. Sometimes there can be a null value as well. Only the A and D values will correspond to an Active and reliable Sentence. This mode character has been added to the RMC, RMB, VTG, and GLL, sentences and optionally some others including the BWC and XTE sentences.

Software Kompatibilitet

NMEA 0183 is supported by various navigation and mapping software. Notable applications include:

  • DeLorme Street Atlas
  • ESRI
  • Google Earth
  • Google Maps Mobile Edition[5]
  • gpsd - Unix GPS Daemon
  • JOSM - OpenStreetMap Map Editor
  • MapKing
  • Microsoft MapPoint
  • Microsoft Streets & Trips
  • NetStumbler
  • OpenCPN - Open source navigation software
  • OpenBSD's hw.sensors framework with the nmea(4) pseudo-device driver[6]
  • OpenNTPD through sysctl hw.sensors § timedelta API
  • Rand McNally StreetFinder
