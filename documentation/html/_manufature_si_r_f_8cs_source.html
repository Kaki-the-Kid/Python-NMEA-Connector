5  /* SiRF
6  * The SiRF line of chips support several input sentences that permit
7  * the user to customize the way the chip behaves. In addition SiRF has
8  * a binary protocol that is even more powerful permitting different
9  * implementations to behave entirely differently. However, most
10  * applications do not attempt to customize the behavior so a user will
11  * need to make sure that the any customization is compatible with the
12  * application they are planning to use. There are 5 input sentences
13  * defined that begin with $PSRF which is followed by three digits.
14  * Each sentence takes a fix amount of input fields which must exist,
15  * no null fields, and is terminated with the standard CR/LF sequence.
16  * The checksum is required.
17  *
18  * The sentences 100 and 102 set the serial ports. 100 sets the main
19  * port A while 102 sets the DGPS input port B. 100 has an extra field
20  * that can be used to switch the interface to binary mode. Binary mode
21  * requires 8 bits, 1 stop bit, no parity. There is a command in binary
22  * mode that will switch the interface back to NMEA. Do not use the
23  * NMEA command to switch to binary mode unless you have the ability to
24  * switch it back. You could render your gps inoperative.
25  *
26  * $PSRF100,0,9600,8,1,0*0C
27  * $PSRF102,9600,8,1,0*3C
28  *
29  * where:
30  * $PSRF100
31  * 0 0=SiRF, 1=NMEA - This is where the protocol is changed.
32  * 9600 b/s rate 4800, 9600, 19200, 38400
33  * 8 7, 8 Databits
34  * 1 0, 1 Stopbits
35  * 0 0=none, 1=odd, 2=even Parity
36  * *0C checksum
37  *
38  * The sentences 101 and 104 can be used to initialize values to be used
39  * by the gps. Supplying these values can shorten the initial lock time.
40  * If the clock offset is set to 0 then an internal default will be used.
41  * Sentence 101 supplies data in the internal ECEF (Earth centered, Earth
42  * Fixed) format in meters while sentence 104 supplies the data in the
43  * traditional Lat / Lon format.
44  *
45  * $PSRF101,-2686700,-4304200,3851624,95000,497260,921,12,3*22
46  * $PSRF104,37.3875111,-121.97232,0,95000,237759,922,12,3*3A
47  *
48  * where:
49  * $PSRF104
50  * 37.3875111 Latitude in degrees
51  * -121.97232 Longitude in degrees
52  * 0 Ellipsoid Altitude in meters
53  * 95000 Clock offset
54  * 237759 GPS Time of Week in seconds
55  * 922 GPS Week Number
56  * 12 Channel count (1 to 12)
57  * 3 Reset config where
58  * 1 = warm start, ephemeris valid
59  * 2 = clear ephemeris, warm start (First Fix)
60  * 3 = initialize with data, clear ephemeris
61  * 4 = cold start, clear all data
62  * 8 = cold start, set factory defaults
63  * *3A checksum
64  *
65  * The sentence 103 is used to control which NMEA sentences are to be
66  * sent and how often. Each sentence type is controlled individually.
67  * If the query bit is set then the gps responds by sending this message
68  * in the next second no matter what the rate is set to. Note that if
69  * trickle power is in use (can only be set in binary mode) then the
70  * actual update rate will be the selected update rate times the trickle
71  * rate which could mean that the data will be sent less frequently than
72  * was set here.
73  *
74  * $PSRF103,05,00,01,01*20
75  *
76  * where:
77  * $PSRF103
78  * 05 00=GGA
79  * 01=GLL
80  * 02=GSA
81  * 03=GSV
82  * 04=RMC
83  * 05=VTG
84  * 00 mode, 0=set rate, 1=query
85  * 01 rate in seconds, 0-255
86  * 01 checksum 0=no, 1=yes
87  * *20 checksum
88  *
89  * The 105 sentence controls a debug mode which causes the gps to
90  * report any errors it finds with the input data. $PSRF105,1*3E would
91  * turn debug on while $PSRF105,0*3F would turn it off.
92  */
