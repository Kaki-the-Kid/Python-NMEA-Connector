5  /* GGA - essential fix data which provide 3D location and accuracy data.
6  * The most important NMEA sentences include the GGA which provides the
7  * current Fix data, the RMC which provides the minimum gps sentences
8  * information, and the GSA which provides the Satellite status data.
9  *
10  * $GPGGA,123519,4807.038,N,01131.000,E,1,08,0.9,545.4,M,46.9,M,,*47
11  *
12  * where:
13  * GGA Global Positioning System Fix Data
14  * 123519 Fix taken at 12:35:19 UTC
15  * 4807.038,N Latitude 48 deg 07.038' N
16  * 01131.000,E Longitude 11 deg 31.000' E
17  * 1 Fix quality: 0 = invalid
18  * 1 = GPS fix (SPS)
19  * 2 = DGPS fix
20  * 3 = PPS fix
21  * 4 = Real Time Kinematic
22  * 5 = Float RTK
23  * 6 = estimated (dead reckoning) (2.3 feature)
24  * 7 = Manual input mode
25  * 8 = Simulation mode
26  * 08 Number of satellites being tracked
27  * 0.9 Horizontal dilution of position
28  * 545.4,M Altitude, Meters, above mean sea level
29  * 46.9,M Height of geoid (mean sea level) above WGS84
30  * ellipsoid
31  * (empty field) time in seconds since last DGPS update
32  * (empty field) DGPS station ID number
33  * *47 the checksum data, always begins with *
34  *
35  * If the height of geoid is missing then the altitude should be suspect.

44  /* GSA - GPS DOP and active satellites.
45  * This sentence provides details on the nature of the fix. It includes
46  * the numbers of the satellites being used in the current solution and
47  * the DOP. DOP (dilution of precision) is an indication of the effect
48  * of satellite geometry on the accuracy of the fix. It is a unitless
49  * number where smaller is better. For 3D fixes using 4 satellites a 1.0
50  * would be considered to be a perfect number, however for overdetermined
51  * solutions it is possible to see numbers below 1.0.
52  *
53  * There are differences in the way the PRN's are presented which can
54  * effect the ability of some programs to display this data. For example,
55  * in the example shown below there are 5 satellites in the solution and
56  * the null fields are scattered indicating that the almanac would show
57  * satellites in the null positions that are not being used as part of
58  * this solution. Other receivers might output all of the satellites used
59  * at the beginning of the sentence with the null field all stacked up at
60  * the end. This difference accounts for some satellite display programs
61  * not always being able to display the satellites being tracked. Some
62  * units may show all satellites that have ephemeris data without regard
63  * to their use as part of the solution but this is non-standard.
64  *
65  * $GPGSA,A,3,04,05,,09,12,,,24,,,,,2.5,1.3,2.1*39
66  *
67  * where:
68  * GSA Satellite status
69  * A Auto selection of 2D or 3D fix (M = manual)
70  * 3 3D fix - values include: 1 = no fix
71  * 2 = 2D fix
72  * 3 = 3D fix
73  * 04,05... PRNs of satellites used for fix (space for 12)
74  * 2.5 PDOP (dilution of precision)
75  * 1.3 Horizontal dilution of precision (HDOP)
76  * 2.1 Vertical dilution of precision (VDOP)
77  * *39 the checksum data, always begins with *
78  */

83  /* GSV - Satellites in View
84  * shows data about the satellites that the unit might be able to find
85  * based on its viewing mask and almanac data. It also shows current
86  * ability to track this data. Note that one GSV sentence only can provide
87  * data for up to 4 satellites and thus there may need to be 3 sentences
88  * for the full information. It is reasonable for the GSV sentence to contain
89  * more satellites than GGA might indicate since GSV may include satellites
90  * that are not used as part of the solution. It is not a requirment that
91  * the GSV sentences all appear in sequence. To avoid overloading the data
92  * bandwidth some receivers may place the various sentences in totally
93  * different samples since each sentence identifies which one it is.
94  *
95  * The field called SNR (Signal to Noise Ratio) in the NMEA standard is often
96  * referred to as signal strength. SNR is an indirect but more useful value
97  * that raw signal strength. It can range from 0 to 99 and has units of dB
98  * according to the NMEA standard, but the various manufacturers send different
99  * ranges of numbers with different starting numbers so the values themselves
100  * cannot necessarily be used to evaluate different units. The range of working
101  * values in a given gps will usually show a difference of about 25 to 35 between
102  * the lowest and highest values, however 0 is a special case and may be shown on
103  * satellites that are in view but not being tracked.
104  *
105  * $GPGSV,2,1,08,01,40,083,46,02,17,308,41,12,07,344,39,14,22,228,45*75
106  *
107  * where:
108  * GSV Satellites in view
109  * 2 Number of sentences for full data
110  * 1 sentence 1 of 2
111  * 08 Number of satellites in view
112  * 01 Satellite PRN number
113  * 40 Elevation, degrees
114  * 083 Azimuth, degrees
115  * 46 SNR - higher is better
116  * for up to 4 satellites per sentence
117  * *75 the checksum data, always begins with *
118  */
